# 概要
slackに画像を投稿したら自動で特定のGoogle Driveフォルダに保存するslack bot

# 背景・目的
学生エンジニアサークルでslackを使用している。
飲み会やイベントの画像をslackで共有しあう文化がある。
その画像はサークル紹介や活動報告として使うことがある。
しかしフリープランなため3ヶ月で履歴が消えてしまう。

>>> Google Driveに自動で保存することで長期保存したい

# 要件定義
- slackのpublicチャンネルに画像を投稿するとGoogle Driveフォルダに保存される

# 機能要件
- このbotが入れられているslackのpublic channelの投稿を受け取り、投稿内に画像が存在するか検証する機能
  - DMやprivate channelは考慮しない。
- 画像をGoogle Driveに保存する機能
  - 保存先のフォルダのデフォルト値は「年/チャンネル名」
    - 「年」は「年代」 + 「作成日」とする
      - 年代は、毎年10/1に更新されるものとする。
      - 年代は、現在「45th」。2026/10/1以降は「46th」になるということ
      - 作成日とは、年代の更新日のことで、すなわち10/1
      - 例
        - 2026/9/30のチャンネルAのデフォルト値は「45th(2025/10/1~)/チャンネルA/」となる。
        - 2026/10/1のチャンネルAのデフォルト値は「46th(2026/10/1~)/チャンネルA/」となる
      - フォルダが存在しない場合は新規作成する
  - 保存するのは画像のみ。ファイル(PDF等)は保存しない(slack botが画像とファイルを区別してるか知らんけどいい感じにして)
  - ファイル名は適当でOK。
  - 余計なメタデータ(投稿者や投稿時間、投稿chとか諸々)への操作は一切不要なのでデフォルトのままで良い。気になったら都度直すので。

# 非機能要件
- 処理速度とか気にしてない。画像がちゃんと保存されるならなんでもいい
- 学生団体で運用する予定
  - 原則無料でシステム構築すること。クレジットカード登録も避ける。
  - テストコードは不要
  - なるべく簡単な構成・コードにする。

# 技術選定
- Google App Script
  - claspを使ってgitでコード管理可能にする
  - Cloud Run 関数はクレカ登録必要なので保守できなくて1世代で使用不可になる
  - DBや複雑な拡張を行うほどの労力かけるものではないのでGASで十分
